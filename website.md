# Webサイト設計図: Surf Nomad Roast

## 1. ブランド・コンセプト
*   **ブランド名**: **Surf Nomad Roast** (サーフ・ノマド・ロースト)
*   **メイン・キャッチコピー**: **Stillness in every sip.** (この一口に、静寂を。)
*   **ステートメント**: 「良い波とうねり（Swell）を求め、旅を続ける放浪者のためのコーヒー。騒がしい日常を離れ、海辺で一人、自分と自分自身が静かに重なり合う時間を。」
*   **目的**: 湘南のサーファーをメインターゲットとした、ファンの獲得とコーヒー豆のオンライン販売。
*   **キーワード**: 海, リラックス, チル, ヒッピー, 静寂, 自然回帰.
*   **ターゲット**: サーファー、現実から離れたい人、自然回帰に興味がある人。
*   **雰囲気**: 海を感じさせ、落ち着いた「静寂」と「リラックス」が共存するデザイン。

## 2. ページ構成 (スモールスタート案)
初心者でも作りやすく、メンテナンスしやすいシンプルなHTML構成とします。

### ホーム (index.html)
*   **ヒーローエリア**: 洗練された海の画像背景に「Stillness in every sip.」の文字を配置。最大限の「静寂」を表現するため、トップページはヒーローエリアのみのミニマルな構成とします。

### コーヒー豆 (products.html)
*   **プロダクトリスト**:
    *   [Morning Swell]: 朝の波のようにクリアな味わい。
    *   [Nomad Spirit]: どこか懐かしく深いコク。
    *   [Stillness Horizon]: 穏やかな海のような余韻。
*   ※各商品から外部販売サイト（BASE等）へリンク。

### 私たちのストーリー (about.html)
*   **コア・メッセージ**: 「都会の忙しさから離れ、自分自身の心地よさを取り戻す。」
*   **内容**: 効率や数字を追い求める日々の中で見失いがちな「自分だけの時間」を、海辺のコーヒーを通じて取り戻すことの提案。
*   **構成**:
    1.  都会の忙しないリズムへの葛藤。
    2.  誰かの決めたルールではなく、自分の心に従う時間。
    3.  一杯のコーヒーがもたらす「静寂」と豊かな人生。

### コンタクト (contact.html)
*   **案内**: 質問はこちらから。SNSでの日々の風景のシェア。

## 3. デザイン要素
### カラーパレット (案：透き通るような静寂)
*   **ベース (Airy Sand)**: #FAF9F6 (明るく清潔感のあるオフホワイト) - サイト全体の明るさと広がり。
*   **メイン (Morning Mist Blue)**: #B0C4DE (朝靄のような淡いブルー) - 静寂、透明感。暗すぎない爽やかさ。
*   **アクセント (Sun-kissed Wood)**: #C19A6B (日に焼けた流木のベージュ) - ヒッピー、自然、コーヒーの温かみ。
*   **テキスト (Soft Charcoal)**: #333333 (柔らかな墨色) - 真っ黒ではない、目に優しい色。

### 画像/ビジュアルの方向性
*   **ヒーロー画像**: 光が差し込む透明度の高い朝の海、または白い砂浜に置かれたコーヒー。
*   **質感**: 「明るい白」を基調とし、余白を贅沢に使うことで「静寂」を表現します。
*   **装飾**: 繊細な細いラインや、柔らかな光の反射（レンズフレアなど）。

## 7. セキュリティ・安全性の設計 (Security & Safety)
初心者が安心してインターネットに公開（GitHub Pages等）できるよう、以下のルールを徹底します。

### 7.1 静的サイトの「攻めの防御」
*   **サーバーを持たない**: データベースやサーバーサイド・プログラムがないため、SQLインジェクションなどの一般的な攻撃対象になり得ません。これが最大の防御です。
*   **情報の持たない**: 顧客情報や注文情報は、自社サイト内には一切保存せず、信頼できる外部サービス（BASE/STORES 等）にのみ持たせます。

### 7.2 APIキーの管理ルール (重要)
もし将来的に Google Maps 等の API キーが必要になった場合、純粋な HTML サイトでは「キーを隠す（.env）」ことが原理上できません（コードがブラウザで全て見えるため）。そのため、以下の方法で守ります。

1.  **リファラ制限 (HTTP Referrer Restriction)**:
    *   API の管理画面（Google Cloud 等）で、「自分のドメイン（例: surfnomad.com）以外からのリクエストを拒否する」設定を必ず行います。これにより、キーを盗まれても他人のサイトでは使えなくなります。
2.  **クォータ設定 (使用量制限)**:
    *   万が一の不正利用に備え、一日の最大利用量や予算アラートを設定し、想定外の課金を防ぎます。

### 7.3 Git / GitHub セキュリティ
*   **公開リポジトリの注意**: GitHub の「Public」リポジトリにアップロードしたものは、全世界から閲覧可能です。API キーやパスワードをコード内に書いたままコミット（保存）しないよう、常に意識します。
*   **書き間違いの防止**: もし秘匿情報が必要な高度な開発に移行する場合は、その時点で `.gitignore` の設定やコンパイルツールの導入を検討します。

## 8. 実装基準・シンプルさの維持 (Simple First)
初めてのサイト制作でも迷わず、着実に形にしていけるよう、以下の「引き算の設計」を徹底します。

*   **1. ビルドツール・フレームワークの不使用**:
    *   Vite や React などの複雑な環境構築は行わず、ブラウザでそのまま開ける「純粋な HTML/CSS」のみで完結させます。
*   **2. 静的ファイルの原則**:
    *   データベースやサーバーサイド・プログラム（PHP/Node.js等）を一切使わず、メンテナンスコストをゼロにします。
*   **3. 外部サービスの活用**:
    *   「ショッピングカート」や「お問い合わせフォーム」など、実装難易度とセキュリティリスクが高い機能は、BASE や Instagram DM などの信頼できる外部サービスへリンクさせるだけに留めます。
*   **4. デザイン優先・機能最小限**:
    *   複雑なスライダーやアニメーションよりも、**「心地よい余白」「美しい文字配置」「透明感のある画像」**という、ブランドの核となる視覚要素の磨き上げを最優先します。
